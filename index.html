Methodology

The feed is a credibility surface, not a scoreboard. It shows how PropellerSwap reached a route decision when using Fynd over Tycho state, and separates evidence available today from future optimality claims.

Benchmark Cycle

A production event should be emitted when PropellerSwap receives a CoW auction, computes a Fynd route, submits or skips, and records the auction result. The dashboard then renders the event as an evidence trail, not as a private internal log.

Route Inspection

Expanded rows show candidate routes Fynd evaluated during the solve, the chosen path, estimated output, gas, net score, and the algorithm label that produced each candidate.

What Is Verifiable

CoW Explorer verifies auction outcomes. Etherscan verifies settlement transactions for wins. Fynd's public routing logic makes candidate evaluation inspectable instead of asking the reader to trust a black-box router.

Current Simulation Boundary

Only the auction events are simulated in this PoC. The Tycho proxy, Fynd build, local server, event ingestion endpoint, filtering, route expansion, and export path are production-shaped.

Why This Is Different

The product does not ask users to believe PropellerSwap is good because a chart says so. It lets them inspect the actual route decision pattern behind each visible auction.

Run It Yourself

Stream live CoW orders, solve them locally with Fynd, then modify the algorithm and compare your route decisions against PropellerSwap's public feed. This is solve-only; submitting settlements requires being a registered CoW solver.

Local Loop

Start Fynd locally, subscribe to live CoW orders, run the same route search, and compare candidate routes, scores, and selected paths against the production feed.

Experiment Surface

Change algorithm weights, route depth, pool filters, gas assumptions, or worker-pool strategy. The point is not to trust the dashboard, but to see whether your machine can find a better route.
